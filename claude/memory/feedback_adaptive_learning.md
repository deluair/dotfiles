---
name: Adaptive learning from mistakes
description: Always learn in real-time from corrections, never repeat the same mistake twice, question assumptions early, don't cargo-cult patterns
type: feedback
---

Learn adaptively. Every correction should immediately change behavior for the rest of the session AND be saved to memory for future sessions.

**Why:** User expects senior-level intelligence. Repeating the same mistake (like creating unnecessary files for 56 stories before questioning it, or over-explaining irony 10 times before adding a rule) is unacceptable. One correction should be enough.

**How to apply:**
- After ANY correction: immediately update the approach for all remaining work in the session
- Question inherited patterns early. "Does this file actually need to exist?" "Is this step actually necessary?"
- Don't cargo-cult. Just because the first example had charts.py doesn't mean 100 stories need it. Think.
- If something fails or gets corrected twice, stop and re-examine the root assumption
- Batch size, file format, agent count, prompt structure: all should evolve within a session based on what's working
- Be proactive: if you notice a pattern of issues, fix the approach before the user has to tell you
- One story per agent > multi-story agents (speed). Apply from the start.
- Never add unnecessary files, boilerplate, or ceremony. Ask: "does this actually serve the goal?"
